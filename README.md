# Spatial-Temporal GNN for Regional Temperature Forecasting with DLOps
---

## Overview

This project implements a **Spatial-Temporal Graph Neural Network (ST-GNN)** for multi-horizon temperature forecasting across eight weather stations in the Atlanta, Georgia metropolitan area. The model combines:

- **GCN layers** — exclusively for spatial correlation learning across stations
- **GRU layers** — exclusively for temporal dynamics across the 24-hour input window

The full pipeline is production-ready with a **DLOps framework** integrating Apache Airflow, DVC, and MLflow for orchestration, versioning, and experiment tracking.

## Architecture

### ST-GNN Model

```
Input: X ∈ R^(B × T × N × F)
  B = batch size
  T = 24 (input window hours)
  N = 8 (number of stations)
  F = 11 (features per station)

Spatial Encoder (GCN):
  → GCN Layer 1 (GELU activation)
  → GCN Layer 2 + Residual Projection + LayerNorm
  → Dropout (p=0.55)

Temporal Encoder (GRU):
  → 2-layer GRU (hidden_dim=32)
  → Final hidden state

Output Heads:
  → 4 linear heads (one per horizon: 1h, 6h, 12h, 24h)
  → Output: ŷ ∈ R^(N × 4)
```

### Graph Construction

- **Method**: Distance-weighted K-Nearest Neighbor (k=5)
- **Distance**: Haversine great-circle formula
- **Edge weights**: Exponential decay — `a_ij = exp(-d_ij / σ)`, σ=120km
- **Normalization**: Symmetric GCN — `Â = D̃^(-1/2) Ã D̃^(-1/2)`

## Running the Pipeline

### Option 1: DVC (Recommended)

Run the full pipeline end-to-end with a single command:

```bash
# Run all stages: ingest → preprocess → train → evaluate
dvc repro

# View pipeline DAG
dvc dag
```

DVC automatically tracks which stages need to re-run based on changed inputs.

### Option 2: Run Stages Manually

```bash
# Stage 1: Collect data from Open-Meteo API
python -m src.data_ingestion

# Stage 2: Preprocess, normalize, build graph
python -m src.preprocess

# Stage 3: Train ST-GNN and LSTM baseline
python -m src.train

# Stage 4: Evaluate and generate plots/tables
python -m src.evaluate
```

### Option 3: Apache Airflow

```bash
# Set Airflow home
export AIRFLOW_HOME=$(pwd)/airflow_home

# Initialize Airflow
airflow db init

# Start Airflow webserver and scheduler
airflow webserver --port 8080 &
airflow scheduler &

# Trigger the pipeline DAG
airflow dags trigger weather_stgnn_pipeline
```

Access the Airflow UI at `http://localhost:8080`.

## Hyperparameters

All hyperparameters are controlled via `params.yaml`:

```yaml
data:
  start_date: "2025-01-01"
  end_date: "2026-03-31"
  input_window: 24          # Hours of historical input
  horizons: [1, 6, 12, 24] # Forecast horizons in hours
  target_col: temperature_2m
  train_ratio: 0.70
  val_ratio: 0.15

graph:
  method: distance_weighted_knn
  k_neighbors: 5
  distance_sigma_km: 120

model:
  hidden_dim: 32
  temporal_layers: 2
  dropout: 0.55

train:
  epochs: 15
  batch_size: 32
  learning_rate: 0.0003
  weight_decay: 0.0001
  patience: 2              # Early stopping patience
```

## Data Source

All weather data is collected from the **Open-Meteo API** — free, no API key required:
- URL: https://open-meteo.com/
- Variables: temperature (2m), relative humidity, wind speed, wind direction, surface pressure
- Resolution: Hourly
- Period: January 2025 – March 2026

---

## Features

Each station has F=11 features per timestep:
- 5 meteorological: `temperature_2m`, `relative_humidity_2m`, `wind_speed_10m`, `wind_direction_10m`, `surface_pressure`
- 6 cyclical time encodings: `hour_sin/cos`, `day_of_week_sin/cos`, `day_of_year_sin/cos`
